🪪 ELID API 문서

ELID 디지털 명함 서비스의 공식 REST API 문서입니다.
모든 API는 JSON 형식으로 요청/응답하며, JWT Bearer 토큰 기반 인증을 사용합니다.

v1.0.0 Base URL: /api/v1 JWT Bearer Auth

🌐 Base URL

/api/v1

🔐 인증 방식

JWT Bearer Token
Authorization: Bearer {token}

📦 콘텐츠 타입

Content-Type:
application/json

⏰ 토큰 만료

Access Token: 7일
Refresh Token: 30일

🔐인증 방식
대부분의 API는 Authorization 헤더에 Bearer 토큰이 필요합니다.
토큰은 로그인(POST /auth/login) 또는 토큰 갱신(POST /auth/refresh)으로 발급받습니다.
Request Header
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... Content-Type: application/json
JWT Payload 구조
{ "sub": "123", // 유저 ID (string) "email": "user@example.com", "plan": "free", // free | pro | business "account_type": "personal", // personal | headhunter "iat": 1714118400, "exp": 1714723200 }
파트너 API 인증 (별도)
X-Partner-API-Key: meti_partner_abc123def456... // 관리자가 발급한 파트너 전용 API 키
📐공통 응답 형식 & 에러 코드
성공 응답 (200 / 201)
{ "success": true, "data": { /* 결과 데이터 */ }, "message": "처리 완료" }
페이지네이션 응답
{ "success": true, "data": [ /* 항목 배열 */ ], "pagination": { "page": 1, "limit": 20, "total": 150, "total_pages": 8, "has_next": true, "has_prev": false } }
실패 응답
{ "success": false, "error": "에러 메시지" }
HTTP 코드의미주요 원인
200성공요청 정상 처리
201생성됨리소스 생성 성공
400잘못된 요청유효성 검사 실패, 파라미터 누락
401인증 실패토큰 없음, 만료, 잘못된 자격증명
403권한 없음플랜 제한, 그룹/어드민 권한 부족
404찾을 수 없음리소스 미존재 또는 삭제됨
409충돌이메일 중복, 이미 가입된 그룹 등
500서버 오류DB 오류, 예외 처리
📄페이지네이션
목록 조회 API는 모두 공통 쿼리 파라미터로 페이지네이션을 지원합니다.
파라미터타입기본값설명

신규 회원가입 (이메일 + 비밀번호)
Public
▼
Request Body
필드타입필수설명
