## PATCH 1 — §3 플랜 정책 전체 교체

**v2.4의 §3 전체를 아래로 교체하세요.**

### 3-1. 플랜 정의

> ~~v2.4: Free는 그룹 가입만 가능, Pro는 생성·운영 가능하나 관리자 기능 없음, Business만 그룹 관리자 기능 허용~~
>
> **v2.5 변경**: 기능 제한 방식 폐지.  
> **모든 플랜에서 그룹 생성·운영·관리 기능 동일하게 제공.**  
> 플랜 간 유일한 차이는 **그룹당 최대 멤버 수**뿐이다.

| 플랜 | 구독료 | 구독 지급 포인트 | 명함 한도 | 그룹 생성 | 그룹 관리 | **그룹 최대 멤버 수** |
|------|-------|--------------|---------|:---:|:---:|:---:|
| `free` | 0원 | 0 P | 3개 | ✅ | ✅ | **2명** |
| `pro` | 미정 | 10,000 P/월 | 10개 | ✅ | ✅ | **10명** |
| `business` | 미정 | 500,000 P/월 | 무제한 | ✅ | ✅ | **무제한** |

> - 구독료·포인트·명함 한도·**그룹 최대 멤버 수**는 슈퍼어드민에서 변경 가능 (`plan_configs`)
> - `max_group_members = NULL` → 무제한 (business 기본값)

### 3-2. 멤버 한도 초과 API 응답 (신규)

멤버 가입 신청(`POST /groups/:id/join`), 초대 가입(`POST /auth/invite/:token/join`),  
멤버 승인(`PATCH /groups/:id/members/:userId` action=approve) 시  
서버가 **그룹 관리자 플랜의 멤버 한도** 를 자동 검사한다.

한도 초과 시 응답:
```json
{
  "success": false,
  "error": "플랜 멤버 한도에 도달했습니다. 플랜을 업그레이드해주세요.",
  "error_code": "plan_member_limit_reached",
  "current": 2,
  "limit": 2,
  "upgrade_required": true
}
```

> `upgrade_required: true` 수신 시 → PATCH 11 §14-4 UX 가이드 적용

### 3-3. 플랜별 명함 한도 초과 처리 (v2.4에서 변경 없음)

```json
{ "success": false, "error_code": "card_limit_exceeded", "upgrade_required": true }
```
→ `upgrade_required: true` 수신 시 업그레이드 유도 다이얼로그 표시

### 14-4. 멤버 한도 초과 UX 가이드 (신규)

**발생 시점**: 그룹 관리자가 멤버 초대·가입 승인 시 플랜 한도 초과  
**API 오류 코드**: `plan_member_limit_reached`

**처리 흐름**:
```
PATCH /groups/:id/members/:userId { action: "approve" }
  또는 POST /groups/:id/join
  또는 POST /auth/invite/:token/join
     ↓
응답: { error_code: "plan_member_limit_reached", limit: 2, upgrade_required: true }
     ↓
모달 또는 바텀시트 표시:

  ┌─────────────────────────────────────┐
  │  📋 멤버 한도에 도달했습니다           │
  │                                     │
  │  현재 Free 플랜은 그룹당 최대          │
  │  2명까지 관리할 수 있습니다.           │
  │                                     │
  │  Pro로 업그레이드하면                 │
  │  최대 10명까지 초대할 수 있어요!       │
  │                                     │
  │     [Pro 구독하기]    [나중에]        │
  └─────────────────────────────────────┘

[Pro 구독하기] → 인앱결제 구독 화면
```

> ⚠️ **필수 준수**: "웹에서 결제하세요" 등 외부 결제 유도 문구 사용 금지 (Apple/Google 정책)
